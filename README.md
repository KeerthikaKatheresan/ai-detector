# AI Writing Detector

Detects whether a piece of text was written by a human or generated by AI.
Built with Python, Streamlit, and scikit-learn.

---

## Project Structure

```
ai_detector/
├── app.py                  ← Streamlit UI (run this)
├── train_model.py          ← Train & save the model (run once)
├── feature_extractor.py    ← All feature engineering logic
├── model.pkl               ← Saved model (created after training)
├── requirements.txt
└── README.md
```

---

## Setup (VS Code)

### 1. Create & activate virtual environment

```bash
# In VS Code terminal (Ctrl + `)
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Train the model (run once — ~5 mins)

```bash
python train_model.py
```

This downloads the HC3 dataset (~50MB), extracts features,
trains a Gradient Boosting model, and saves `model.pkl`.
Expected accuracy: ~85–90%.

### 4. Launch the app

```bash
streamlit run app.py
```

Opens at: http://localhost:8501

## How It Works

### Features extracted per text (9 total)

| Feature | Signal |
|---|---|
| Burstiness (CV of sentence lengths) | High = Human |
| Shannon Entropy | High = Human |
| Type-Token Ratio | High = Human |
| Avg Sentence Length | High = AI |
| AI Phrase Count | High = AI |
| Hedge Adverb Count | High = AI |
| Contraction Count | High = Human |
| First-Person Count | High = Human |
| Word Count | Normalizing |

### Model
- Algorithm: Gradient Boosting Classifier (scikit-learn)
- Dataset: HC3 (Hello-SimpleAI/HC3) — human QA vs ChatGPT answers
- Pipeline: StandardScaler → GradientBoostingClassifier

### Verdict thresholds

| AI Probability | Verdict |
|---|---|
| ≥ 70% | Likely AI-Generated |
| 50–69% | Possibly AI-Generated |
| 35–49% | Uncertain |
| < 35% | Likely Human-Written |
